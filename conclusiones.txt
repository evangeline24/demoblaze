## **ARQUITECTURA IMPLEMENTADA:**
-----------------------------
**• PATRÓN BDD (Behavior Driven Development):**
   - Feature files en lenguaje Gherkin para especificaciones ejecutables
   - Escenarios comprensibles por stakeholders técnicos y no técnicos
   - Reportes HTML que muestran comportamiento real vs esperado

**• KARATE FRAMEWORK:**
   - Sintaxis simplificada para pruebas API sin boilerplate code
   - Validaciones integradas para JSON, XML, texto plano
   - Manejo automático de HTTP headers y content types
   - Reportes detallados con requests/responses completos

**• BUENAS PRÁCTICAS DE PRUEBAS API:**
   - Generación dinámica de datos de prueba
   - Validación de status codes y response bodies
   - Manejo de casos positivos y negativos
   - Logging detallado para debugging

## **HALLADOS DURANTE LA AUTOMATIZACIÓN:**
--------------------------------------
**OBSERVACIONES POSITIVAS:**
   - La API de Demoblaze es consistente y confiable
   - Los mensajes de error son descriptivos y útiles
   - Los endpoints responden dentro de tiempos aceptables
   - La estructura de respuestas es predecible

**DETALLES ENCONTRADOS EN LA API:**
   - Registro exitoso: Retorna string vacío "" (no objeto JSON)
   - Error usuario duplicado: "This user already exist." (typo en "exist")
   - Login exitoso: Token en formato "Auth_token: [valor_base64]"
   - Error contraseña: "Wrong password." (punto incluido)
   - Error usuario inexistente: "User does not exist."
   - Todos los errores retornan status 200 (no 4xx)

## **DESAFÍOS SUPERADOS:**
----------------------
**CONFIGURACIÓN DE DEPENDENCIAS:**
   - Confusión entre karate-junit5 y karate-apache
   - **SOLUCIÓN:** Uso exclusivo de karate-junit5 1.4.1
   - **RESULTADO:** Build exitoso con Maven

**GENERACIÓN DE DATOS ÚNICOS:**
   - Necesidad de usuarios únicos entre ejecuciones
   - **SOLUCIÓN:** JavaScript + timestamp + random number
   - **RESULTADO:** Datos únicos sin conflictos

**VALIDACIÓN DE RESPONSE BODIES:**
   - Respuesta vacía en registro exitoso
   - **SOLUCIÓN:** match response == "" (string vacío)
   - **RESULTADO:** Validación precisa del comportamiento

**MANEJO DE VERSIONES DE KARATE:**
   - karate.random() no disponible en versión 1.4.1
   - **SOLUCIÓN:** Math.random() de JavaScript nativo
   - **RESULTADO:** Funcionalidad equivalente lograda
